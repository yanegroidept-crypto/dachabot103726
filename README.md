# 🏡 DachaBot (ДачаБот)

**DachaBot** — это автоматизированная система публикаций для дачного Telegram-чата садового товарищества (СНТ). Проект построен на концепции «цифровой семьи», где шесть уникальных ботов-персонажей с прописанными характерами, манерой речи и родственными связями ведут интерактивное общение, делятся профессиональными советами по садоводству, прогнозом погоды и дачным юмором.

Система работает автономно на базе **GitHub Actions**, используя **Gemini API (Google GenAI)** для генерации и адаптации текстов и **OpenWeatherMap API** для получения прогноза погоды. Состояние системы (история чата, использованные шутки, прогресс тем) сохраняется напрямую в репозитории через автоматические коммиты.

## ✨ Основные особенности и технологии

### 1. 🔄 Ротация API-ключей Gemini и цепочка моделей (Fallback Chain)
Для обхода жестких лимитов бесплатного тарифа Gemini (Rate Limits) и обеспечения 100% отказоустойчивости реализован продвинутый клиент (`src/gemini_client.py`):
* **Ротация 10 ключей:** Система принимает переменные окружения от `GEMINI_KEY_1` до `GEMINI_KEY_10`. Индекс текущего ключа сохраняется в `state/gemini_key_index.json` и сдвигается на `+1` после **каждого** запроса (успешного или нет). Это гарантирует, что при временной блокировке ключа следующая попытка пойдет с нового.
* **Каскад моделей (Fallback Chain):** При запросе система последовательно пробует 3 модели по убыванию их «силы» и стоимости:
  1. `gemini-3.1-pro-preview` (основная, самая умная модель для сложной стилизации).
  2. `gemini-3.5-flash` (быстрая и качественная альтернатива при исчерпании лимитов первой).
  3. `gemini-3.1-flash-lite` (легковесный резервный вариант).
* На каждую модель выделяется до `5` попыток с задержкой в `2` секунды и ротацией ключей.
* **Отказоустойчивый парсинг:** Если модель возвращает JSON с ошибками или обычный текст, система автоматически очищает markdown-разметку (```json...```) и берет сырой ответ как текст поста, не прерывая выполнение.

### 2. 🌤️ Умная погодная интеграция
Скрипт `src/weather.py` запрашивает детальный 5-дневный прогноз погоды с шагом 3 часа по географическим координатам СНТ через OpenWeatherMap API:
* **Утренний пост (08:30):** Извлекает диапазон температур на сегодня, репрезентативное описание погоды ближе к полудню, скорость ветра и влажность.
* **Вечерний пост (21:45):** Извлекает подробный прогноз на завтра, а также аккуратно форматирует краткую сводку по остальным дням до конца недели (с учетом дней недели и дат).
* **Резервные сценарии:** Предусмотрен автоматический откат на текущую погоду через эндпоинт `/weather`, если прогнозные данные по какой-то причине недоступны, а также подготовлена альтернативная закомментированная реализация на базе полностью бесплатного API **Open-Meteo** (не требующего ключей).

### 3. 📚 Управление стеком тем (Topics Stack)
Для публикаций в топик «Дача, сад, огород» используется готовый стек качественных статей в JSON-формате (`topics_stack/`):
* Скрипт `src/garden_post_common.py` берёт первую неиспользованную тему, публикует её и помечает `used: true`.
* Когда все темы в файле исчерпаны, стек автоматически сбрасывает все флаги в `false` и запускается по кругу.
* **Адаптация Gemini:** Готовая статья может опционально (флаг `USE_GEMINI_ADAPTATION=true`) прогоняться через Gemini для лёгкой адаптации под стиль персонажа (Нины Васильевны или Петровича) и историю чата, сохраняя при этом оригинальную разметку, структуру заголовков и все исходные эмодзи.

### 4. 🖼️ Синхронизированный «Дачный юмор»
Скрипт `src/post_humor.py` последовательно публикует по одной случайной картинке от лица трёх персонажей (Васильевна, Петрович, Эдик) в специальный топик юмора:
* Список отправленных изображений сохраняется в `state/used_humor.json` для исключения повторов.
* Все три персонажа обрабатываются **внутри одного скрипта и одного workflow**, что полностью исключает конфликты параллельного пуша (Git Race Conditions) в GitHub Actions, которые неизбежно возникали бы при раздельных запусках.

### 5. 🛡️ Безопасность и Best-Effort доставка (`run_safely.py`)
Все основные скрипты обернуты в обработчик `run_safely`:
* Любое непредвиденное исключение логируется с полным трейсбэком.
* Краткая сводка об ошибке записывается в `state/errors.log`.
* Если заданы переменные `ADMIN_BOT_TOKEN` и `ADMIN_CHAT_ID`, администратор мгновенно получает уведомление в Telegram о упавшем посте.
* Скрипты завершаются с **кодом возврата 0**. Это сделано намеренно: временные сбои внешних API (Gemini/Weather) не должны приводить к «красным» статусам воркфлоу в GitHub Actions и спамить на почту владельца репозитория.

## 🛠️ Настройка и развертывание

### Локальное тестирование
1. Установите Python версии `3.12` или выше.
2. Клонируйте репозиторий и установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
3. Скопируйте файл `.env.example` в `.env`:
   ```bash
   cp .env.example .env
   ```
4. Заполните переменные в `.env` своими реальными значениями (см. описание ниже).
5. Запустите любой интересующий скрипт вручную:
   ```bash
   cd src
   python post_morning.py
   ```

### Переменные окружения (`.env` и GitHub Secrets)

Для работы бота требуются следующие токены и параметры. При работе в **GitHub Actions** все эти параметры должны быть занесены в **Repository Secrets** репозитория.

#### 🤖 Токены Ботов Telegram
У каждого бота должен быть свой токен, полученный у `@BotFather`:
* `TG_BOT_TOKEN_VASILEVNA` — токен Нины Васильевны (утро, вечер, сад).
* `TG_BOT_TOKEN_PETROVICH` — токен Петровича (сад, юмор).
* `TG_BOT_TOKEN_EDIK` — токен Эдика (юмор).
* `TG_BOT_TOKEN_OLYA` — токен Ольги (молоко).
* `TG_BOT_TOKEN_TEMA` — токен Тёмы (резерв).
* `TG_BOT_TOKEN_ALESYA` — токен Алеси (резерв).

#### 💬 Параметры чата (Супергруппа Telegram)
Для отправки сообщений боты должны быть добавлены администраторами в супергруппу с включенной функцией тем (Topics):
* `TG_CHAT_ID` — ID супергруппы (обычно начинается с `-100`).
* `TOPIC_ID_GENERAL` — ID топика «Общий чат» (куда идут утренний и вечерний посты).
* `TOPIC_ID_GARDEN` — ID топика «Дача, сад, огород».
* `TOPIC_ID_MILK` — ID топика «Молочная продукция».
* `TOPIC_ID_HUMOR` — ID топика «Дачный юмор».

#### 🔑 Ключи Gemini API (Google GenAI)
Вы можете указать от 1 до 10 ключей. Если у вас меньше 10 ключей, вы можете продублировать один или указать только имеющиеся (скрипт автоматически отфильтрует пустые переменные):
* `GEMINI_KEY_1` ... `GEMINI_KEY_10` — API ключи от Google AI Studio.

#### 🌤️ Настройки погоды (OpenWeatherMap)
* `WEATHER_API_KEY` — API-ключ OpenWeatherMap (бесплатный тариф «Current and Forecast»).
* `DACHA_LAT` — географическая широта вашей дачи (например, `55.7558`).
* `DACHA_LON` — географическая долгота вашей дачи (например, `37.6173`).

#### 🚨 Оповещения администратора (Опционально)
Если хотите получать мгновенные отчеты в случае падения скриптов:
* `ADMIN_BOT_TOKEN` — токен отдельного технического бота для отправки алертов.
* `ADMIN_CHAT_ID` — ваш личный Telegram ID.

## 📊 Форматы файлов состояния и данных

### 1. Стек тем садоводства (`topics_stack/*.json`)
Представляет собой массив объектов. Новые темы берутся сверху (первый элемент с `used: false`).
```json
[
  {
    "id": "blue_tape",
    "used": true,
    "topic": "Синяя изолента как универсальный инструмент дачника",
    "ready_post": "Текст заготовки поста с эмодзи..."
  },
  {
    "id": "plastic_bottle",
    "used": false,
    "topic": "5 способов применения пластиковой бутылки на даче",
    "ready_post": "Текст заготовки..."
  }
]
```

### 2. Краткая история чата (`state/chat_history.json`)
Хранит до 10 последних событий. Подается на вход Gemini при каждом запросе, чтобы боты понимали текущий контекст и не повторялись дословно.
```json
[
  {
    "date": "2026-07-12",
    "author": "Нина Васильевна",
    "summary": "Васильевна написала утренний пост, погода солнечная, +24°C"
  }
]
```

### 3. Лог использованного юмора (`state/used_humor.json`)
Отслеживает, какие картинки из папки `humor/<character_key>/` уже были опубликованы, чтобы избежать спама одной и той же шуткой.
```json
{
  "vasilevna": [
    "photo_5364017306775788439_y.jpg",
    "photo_5364017306775788441_y.jpg"
  ],
  "petrovich": [
    "photo_5364017306775788434_y.jpg"
  ],
  "edik": []
}
```
